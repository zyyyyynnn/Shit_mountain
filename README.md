## 项目定位

这里不是烂代码垃圾桶。

这里收集的是那些**第一眼能跑、第二眼想跑、第三眼开始写事故复盘**的代码。每个展品都要复现一种真实的软件工程反模式，并说明它为什么形成、会如何失控，以及怎样安全拆除。

> This is not a random code dump. It is a runnable museum of code smells, anti-patterns, and refactoring escapes.

| 编号 | 山规 | 解释 |
|---|---|---|
| `SM-01` | 屎要臭得明白 | 必须解释问题所在，不能只靠低级命名制造视觉污染。 |
| `SM-02` | 屎要能够运行 | 展品必须可以验证，或明确声明为预期失败。 |
| `SM-03` | 屎要配铲子 | 提供 `fixed/` 版本，或给出可执行的重构路线。 |
| `SM-04` | 搞怪不能投毒 | 禁止恶意载荷、真实密钥、破坏性脚本与第三方私有代码。 |

## 快速开始

需要 JDK 17 或更高版本，以及 Python 3.10 或更高版本。

```bash
bash scripts/validate.sh
```

查看展品合同，或只运行某一种语言：

```bash
python3 scripts/exhibit_runner.py --list
python3 scripts/exhibit_runner.py --language java
python3 scripts/exhibit_runner.py --language python
```

## 项目如何运作

| 层级 | 责任 |
|---|---|
| `exhibit.json` | 保存标题、摘要、目录元数据、运行入口、超时和行为合同。 |
| `bad/` 与 `fixed/` | 并排保存事故现场与重构结果。 |
| `scripts/exhibit_runner.py` | 在独立临时目录中编译和运行展品，限制时间与输出。 |
| `scripts/smi_engine/` | 按语言提取透明、可回归测试的代码异味指标。 |
| `scripts/docs_generator.py` | 生成完整目录、SMI 文档和固定长度的 README 概览。 |
| GitHub Actions | 在最低支持版本与当前版本上执行完整验山。 |

README 只承担项目首页职责。展品数量、完整排行榜和语言细节分别维护在专用文档中，因此新增展品不会继续拉长首页。

## 贡献展品

标准施工流程：

1. 在 `exhibits/<language>/<number>-<slug>/` 创建独立展品。
2. 在 `exhibit.json` 中声明目录元数据、入口、超时和行为合同。
3. 编写展品自己的 `README.md`，说明案发现场、主要罪名和铲屎路线。
4. 将事故现场放进 `bad/`，修复方案放进 `fixed/`。
5. 运行 `python3 scripts/docs_generator.py --write`。
6. 运行 `bash scripts/validate.sh`。
7. 提交 Pull Request。

完整要求见 [`CONTRIBUTING.md`](CONTRIBUTING.md)。

## 维护哲学

> 展品可以是屎山，仓库本身不能成为屎山。

搞怪属性存在于叙事、命名和展品设计中，不通过牺牲诊断能力、执行安全或维护成本来实现。

## 免责声明

本项目用于教育、娱乐与代码审查训练。请勿把 `bad/` 中的代码复制到生产环境。如果你已经复制了，请至少不要把本仓库写进事故复盘的“参考资料”一栏。
