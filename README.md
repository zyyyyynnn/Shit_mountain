## 🚧 山区公告

本仓库专门收集那些“第一眼能跑、第二眼想跑、第三眼老板问为什么不能改”的代码。

所有展品必须遵守四条山规：

1. **屎要臭得明白**：必须解释坏在哪里，不能只靠变量名恶心人。
2. **屎要能够运行**：默认分支上的展品必须可以编译或明确标记为预期失败。
3. **屎要配铲子**：最好同时提供 `fixed/` 版本或清晰的重构路线。
4. **搞怪不能投毒**：禁止恶意软件、凭证窃取、破坏性脚本、真实密钥和针对真实系统的攻击代码。

## 🏆 屎山指数（Shit Mountain Index）

正在制定中的娱乐性评分：

```text
SMI = 圈复杂度
    + 最大嵌套深度 × 2
    + 重复代码比例
    + 全局变量数量 × 3
    + 魔法数字数量
    + 谎言注释数量 × 5
```

分数越高，不代表越值得模仿；只代表铲屎车需要更大马力。

| 展品 | 主要罪名 | 当前状态 |
|---|---|---|
| `000-hello-shitmountain` | 类名小写、孤零零躺在根目录 | 🗿 镇山之宝 |
| `001-if-else-volcano` | 嵌套判断、魔法值、字符串类型系统 | 🌋 持续喷发 |

## 🧪 本地验山

需要 JDK 17 或更高版本：

```bash
bash scripts/validate.sh
```

检查通过时会看到：

```text
✅ 山体稳定：所有展品至少还能跑。
```

## 🧻 如何贡献一坨优质的屎

请先阅读 [`CONTRIBUTING.md`](CONTRIBUTING.md)。简化版流程：

1. 在 `exhibits/<language>/<number>-<slug>/` 创建独立展品。
2. 提供 `README.md`，说明症状、爆炸方式和重构路线。
3. 将坏代码放进 `bad/`，将修复代码放进 `fixed/`。
4. 运行 `bash scripts/validate.sh`。
5. 提交 PR，并在标题前加上 `[展品]`、`[铲屎]` 或 `[景区建设]`。

欢迎提交荒谬但真实的软件工程反模式。拒绝单纯无法编译、毫无说明、只为冒犯而存在的代码。
