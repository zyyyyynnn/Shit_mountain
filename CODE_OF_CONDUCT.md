# 社区行为准则

本项目讨论的是代码、设计决策和工程环境，不是对开发者人格进行评分。

## 基本原则

- 可以笑代码，不要羞辱写代码的人。
- 批评必须指向可观察的技术问题、约束和后果。
- 不根据经验水平、身份、语言背景或工作经历贬低贡献者。
- 不发布个人信息、私密沟通或未经授权的事故材料。
- 维护者可以编辑、隐藏或移除骚扰、威胁、歧视、灌水和恶意内容。

## Review 方式

推荐：

- “这个共享字段让两个职责发生隐式耦合。”
- “这个案例还缺少可复现的行为合同。”
- “修复版改变了业务结果，需要说明这是缺陷修复还是回归。”

不接受：

- 对作者能力、职业或人格的攻击；
- 用 SMI 分数评价开发者绩效；
- 以搞怪为名发布冒犯性、歧视性或针对个人的内容。

## 处理方式

发现问题时，请在相关 Issue 或 PR 中明确指出行为和影响；涉及骚扰、威胁、隐私或安全风险时，请通过仓库所有者的私密联系方式报告。

维护者可以警告、限制互动、删除内容或禁止参与。处理决定以保护贡献者和项目安全为优先。
